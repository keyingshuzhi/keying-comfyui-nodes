# 视频超分三节点使用手册

适用节点：

- `Keying Video Single Input`
- `Keying Video Super Resolution (Process)`
- `Keying Video Output`

适用场景：

- 单视频超分。
- 不需要在输入节点做预览，只关心处理和导出结果。

## 1. 环境准备

- 将视频放到 `ComfyUI/input/`（可放子目录，例如 `input/video_in/`）。
- 确认模型文件在 `ComfyUI/models/video_vsr/`。
- 系统已安装 `ffmpeg` 和 `ffprobe`，并可在终端直接执行。

## 2. 推荐连线

按顺序连接：

1. `Keying Video Single Input`
2. `Keying Video Super Resolution (Process)`
3. `Keying Video Output`

连线关系：

1. `Single Input.video_paths` -> `Process.video_paths`
2. `Single Input.filenames` -> `Process.source_filenames`
3. `Process.processed_video_paths` -> `Output.processed_video_paths`
4. `Process.source_filenames` -> `Output.source_filenames`

## 3. 节点说明

### 3.1 Keying Video Single Input

作用：

- 从 `input` 目录下拉选择一个视频。
- 输出单视频路径和文件名，供后续节点处理。

参数：

- `single_video_file`：要处理的视频（下拉选择）。

输出：

- `video_paths`：单视频绝对路径（字符串）。
- `filenames`：单视频文件名（字符串）。
- `count`：固定为 `1`。

### 3.2 Keying Video Super Resolution (Process)

作用：

- 执行视频增强流程（时序降噪 + 放大 + 锐化）。
- 中间文件写入 `ComfyUI/temp/<temp_subfolder>/`。
- 节点运行时显示可视化进度条。

关键参数建议：

- `model_name`：从 `models/video_vsr` 选择。
- `scale_hint`：常用 `2` 或 `4`。
- `backend`：保持 `auto`。
- `denoise_strength`：常用 `0.8` 到 `1.5`。
- `temporal_strength`：常用 `1.0` 到 `2.5`。
- `sharpen_strength`：常用 `0.4` 到 `0.9`。
- `output_suffix`：例如 `_vsr`。
- `output_format`：通常 `mp4`。
- `video_codec`：`auto` 或 `libx264`。
- `crf`：画质优先可用 `18-22`，体积优先可用 `23-30`。
- `preset`：`slow` 画质/压缩更优，`ultrafast` 速度更快。
- `keep_audio`：是否保留原音频。
- `overwrite_temp`：是否覆盖临时同名结果。
- `stop_on_error`：单视频建议保持 `true`。

输出：

- `processed_video_paths`：处理后临时视频路径。
- `report`：处理日志。
- `processed_count`：处理成功数量。
- `source_filenames`：传递源文件名供导出节点重命名。
- `preview_image`：处理结果预览帧（可不连接）。

### 3.3 Keying Video Output

作用：

- 将处理后的临时视频导出到 `ComfyUI/output/<subfolder>/`。
- 可选择拷贝或移动文件。

参数：

- `subfolder`：导出子目录，例如 `video_vsr_single`。
- `rename_mode`：
- `keep_processed_name`：使用处理后文件名。
- `use_source_filenames`：使用原始文件名。
- `filename_suffix`：导出文件名附加后缀。
- `overwrite`：是否覆盖同名文件。
- `move_files`：
- `false`：复制到 output（保留 temp 文件）。
- `true`：移动到 output（temp 文件被转移）。
- `stop_on_error`：单视频建议 `true`。

输出：

- `saved_paths`：最终输出文件路径。
- `saved_count`：保存成功数量。
- `preview_image`：导出结果预览帧（可不连接）。

## 4. 单视频推荐配置（可直接用）

- `Process.scale_hint = 4`
- `Process.denoise_strength = 1.2`
- `Process.temporal_strength = 2.2`
- `Process.sharpen_strength = 0.7`
- `Process.output_format = mp4`
- `Process.video_codec = auto`
- `Process.crf = 20`
- `Process.preset = slow`
- `Output.subfolder = video_vsr_single`
- `Output.rename_mode = use_source_filenames`

## 5. 常见问题

`Q1: 下拉里看不到视频`

- 确认文件在 `ComfyUI/input/` 或其子目录。
- 确认扩展名是常见视频格式（`mp4/mov/mkv/avi/webm` 等）。
- 重新加载自定义节点或重启 ComfyUI。

`Q2: 运行时报找不到 ffmpeg`

- 终端执行 `ffmpeg -version` 和 `ffprobe -version` 检查是否可用。
- 若不可用，安装 ffmpeg 并加入 `PATH`。

`Q3: 有进度条但输出目录没有文件`

- 检查 `Output` 节点是否参与执行。
- 查看 `Process.report` 与 `Output` 节点报错。
- 检查 `subfolder` 是否写到了预期目录。
